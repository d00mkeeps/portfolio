## The problem

Transfer pricing is the process of setting prices on transactions between related entities within a corporate group. Get it wrong and you face HMRC scrutiny, adjustments, and penalties. Most companies do this in spreadsheets — which is auditable in theory, but brittle in practice.

The brief was simple: replace a set of Excel models with something that ran automatically, was auditable end-to-end, and produced documentation that a tax adviser could hand directly to HMRC without editing.

## Technical approach

The engine implements three OECD-approved methods:

- **CUP** (Comparable Uncontrolled Price) — direct price comparison
- **TNMM** (Transactional Net Margin Method) — margin benchmarking against comparables
- **Profit Split** — allocation of combined profit based on economic contribution

Each method is its own module with a defined interface: takes transaction data, returns a pricing recommendation plus a confidence interval.

```python
class TNMMEngine:
    def run(self, transactions: pd.DataFrame, comparables: pd.DataFrame) -> PricingResult:
        margin = self._compute_net_margin(transactions)
        benchmark = self._interquartile_range(comparables)
        return PricingResult(
            method="TNMM",
            within_range=benchmark.contains(margin),
            adjustment=benchmark.median - margin if not benchmark.contains(margin) else 0,
        )
```

The outlier detection runs on every transaction at ingest. Anything outside the arm's-length range gets flagged before it hits the pricing layer.

## What I got wrong first

My first cut stored the comparables data as a static CSV baked into the repo. This broke immediately when the client's comparables set updated mid-year. The fix was pulling comparables from a versioned PostgreSQL table with an effective-date column — so you can always reconstruct what the engine would have decided on any given date.

This matters for audits. HMRC wants to see what rate was set *and why*, at the time it was set.

## Outcome

The engine now runs nightly, processes ~3,000 intra-group transactions, and generates structured XML in the OECD CbCR format. The documentation output eliminated approximately 40 hours of manual report-writing per quarter.

The biggest lesson: in regulatory software, auditability is a first-class feature, not an afterthought.
